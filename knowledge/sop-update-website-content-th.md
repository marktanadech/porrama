## โครงสร้างคอลัมน์ใน Google Sheet

ควรรักษาชื่อคอลัมน์เหล่านี้ให้เหมือนเดิม ห้ามเปลี่ยนชื่อหัวตารางโดยไม่แจ้งทีมเทคนิค

| คอลัมน์ | จำเป็นหรือไม่ | วิธีใช้งาน |
| --- | --- | --- |
| `sort_order` | ไม่จำเป็น | ใส่ตัวเลขเพื่อกำหนดลำดับ โครงการเลขน้อยจะแสดงก่อน |
| `visible` | ไม่จำเป็น | ใส่ `yes` เพื่อแสดงบนเว็บ ใส่ `no` เพื่อซ่อน |
| `project_name` | จำเป็น | ชื่อโครงการที่จะแสดงบนการ์ด |
| `project_image` | จำเป็น | URL รูปภาพ หรือ path รูปในเว็บไซต์ เช่น `/images/...` |
| `description` | จำเป็น | คำอธิบายโครงการแบบสั้น กระชับ อ่านง่าย |
| `client` | ไม่จำเป็น | ชื่อลูกค้า |
| `location` | ไม่จำเป็น | สถานที่ของโครงการ |
| `value` | ไม่จำเป็น | มูลค่างาน เช่น `2,323,839 Baht` |
| `duration` | ไม่จำเป็น | ระยะเวลาดำเนินงาน เช่น `3 Months` |
| `category` | ไม่จำเป็น | ประเภทงาน เช่น `Steel structure`, `Civil construction` |
| `year` | ไม่จำเป็น | ปีของโครงการ |

## ขั้นตอนที่ 1: แก้ไขข้อมูลใน Google Sheet

1. เปิด Google Sheet ที่ใช้เป็นฐานข้อมูลโครงการ
2. เลือกแถวของโครงการที่ต้องการแก้ไข
3. แก้ไขข้อมูลเฉพาะคอลัมน์ที่เกี่ยวข้อง
4. หากต้องการให้โครงการแสดงบนเว็บไซต์ ให้ใส่ `yes` ในคอลัมน์ `visible`
5. หากต้องการซ่อนโครงการ ให้ใส่ `no` ในคอลัมน์ `visible`
6. หากต้องการเปลี่ยนลำดับ ให้แก้ตัวเลขใน `sort_order`
7. ตรวจสอบว่า `project_name`, `project_image`, และ `description` มีข้อมูลครบ

ข้อควรระวัง:

- ห้ามลบแถวหัวตาราง
- ห้ามเปลี่ยนชื่อคอลัมน์
- ห้าม merge cell
- หลีกเลี่ยงการใส่บรรทัดว่างจำนวนมากกลางตาราง
- คำอธิบายควรสั้นพอสำหรับแสดงบนการ์ดโครงการ

## ขั้นตอนที่ 2: การใส่รูปภาพโครงการ

รูปภาพสามารถใช้ได้ 2 วิธี

วิธีที่ 1: ใช้รูปที่อยู่ในเว็บไซต์แล้ว

ให้ใส่ path รูปในคอลัมน์ `project_image` เช่น:

```text
/images/profile-projects/stt-parking-shelter.jpg
```

วิธีนี้เหมาะกับรูปที่ดึงมาจาก company profile และถูกเก็บไว้ในระบบเว็บไซต์แล้ว

วิธีที่ 2: ใช้ Google Drive หรือ public image URL

1. อัปโหลดรูปเข้า Google Drive
2. ตั้งค่าแชร์เป็น Anyone with the link can view
3. Copy link ของรูป
4. วาง link ในคอลัมน์ `project_image`

ข้อควรระวัง:

- รูปต้องเปิดดูได้แบบ public
- ควรใช้รูปแนวนอนถ้าเป็นไปได้
- หลีกเลี่ยงรูปที่มีข้อมูลลับหรือโลโก้ลูกค้าที่ไม่ได้รับอนุญาตให้เผยแพร่

## ขั้นตอนที่ 3: ตรวจสอบ Google Sheet ก่อนขึ้นเว็บ

ก่อนตรวจบนเว็บไซต์ ให้เช็กสิ่งเหล่านี้ใน Google Sheet:

- โครงการที่ต้องการให้แสดงมี `visible = yes`
- ทุกแถวที่ visible เป็น `yes` ต้องมีชื่อโครงการ
- ทุกแถวที่ visible เป็น `yes` ต้องมีรูปภาพ
- ทุกแถวที่ visible เป็น `yes` ต้องมีคำอธิบาย
- ลำดับ `sort_order` ไม่ซ้ำกันถ้าต้องการควบคุมลำดับแบบชัดเจน

## ขั้นตอนที่ 4: รอให้ Google Sheet เผยแพร่ข้อมูลล่าสุด

เว็บไซต์อ่านข้อมูลจาก Google Sheet แบบ Published CSV

โดยปกติหลังแก้ Google Sheet:

- รอประมาณ 1-5 นาที
- เปิดเว็บไซต์ใหม่
- กด refresh

หากข้อมูลยังไม่เปลี่ยน:

1. รออีกสักครู่
2. กด refresh อีกครั้ง
3. เปิดเว็บไซต์ใน Incognito mode เพื่อเลี่ยง cache
4. ตรวจว่า Google Sheet ยัง publish อยู่

## ขั้นตอนที่ 6: กรณีต้องกด Redeploy ใน Netlify

การแก้ข้อมูลโครงการใน Google Sheet ปกติไม่จำเป็นต้อง redeploy เพราะเว็บไซต์ดึงข้อมูลจาก Sheet ตอนผู้ใช้เปิดเว็บ

แต่ควรกด redeploy ใน Netlify ถ้ามีกรณีเหล่านี้:

- เพิ่งเพิ่มหรือแก้ environment variable
- เปลี่ยน source code
- เพิ่มรูปภาพใหม่เข้า repository ของเว็บไซต์
- เปลี่ยนหัวเว็บ คำอธิบาย โลโก้ หรือ layout
- เว็บไซต์ไม่อ่าน Google Sheet แม้ URL ถูกต้องแล้ว

วิธี redeploy:

1. เข้า Netlify
2. เลือกเว็บไซต์ Porrama
3. ไปที่เมนู Deploys
4. กด Trigger deploy
5. เลือก Deploy site
6. รอจนสถานะเป็น Published
7. เปิดเว็บไซต์จริงเพื่อตรวจสอบอีกครั้ง
